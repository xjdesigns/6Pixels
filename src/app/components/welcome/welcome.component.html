Welcome to SIX PIXELS

SIX PIXELS was created with the idea to move to an PURE CSS UI frameworks/pattern labs.

Here are the key benefits to using SIX PIXELS

  • Light weight
  • 1 variable file for easy customization
  • IE support for 10 and above
  • Uses no webfonts, only system level fonts
  • Built on an atomic level for minimal trumping
  • All components are scoped to its component
  • Optional includes for components
  • Namespace support(scss compiling required)
  • ZERO JS/External dependancies

Local Development

Below are the commands for running a local instance of the app.


NOTE: You must have gulp installed globally before running


npm install Installs dependencies from our package.json


npm start Starts the local instance and runs our sass with gulp


gulp prod Runs a gulp command for compiling the final minified file

Guidelines

6PIXELS is meant to be a next gen CSS UI FRAMEWORK / PATTERN LAB, and can present progressive enhancement. Please use the @supports. Progressive enhancement code should live at the top of your SCSS file.

Everything should be created with MOBILE FIRST in mind.

Everything should be in increments of 6 PIXELS. This creates specific patterns and expectations.

With any new elements/components please supply a new route, along with at least one live example and code example.
